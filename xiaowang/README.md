# 轻量化刷题网站

一个简单、易用的轻量化刷题系统，支持上传自定义题库、答题练习、错题复习等功能。

## 功能特点

- **自定义题库上传**：支持上传TXT格式的题库文件
- **题库切换**：可管理和切换多个上传的题库
- **题型支持**：支持单选题和多选题
- **自动验证**：答题后自动验证答案正确性
- **错题集**：自动收集错题，支持专门的错题复习模式
- **进度跟踪**：记录学习进度，根据正确率授予不同称号
- **本地存储**：所有数据存储在浏览器本地，无需后端服务器
- **响应式设计**：支持在不同设备上良好显示

## 技术栈

- HTML5
- CSS3
- JavaScript (纯原生)
- LocalStorage API

## 部署到GitHub Pages

### 方法一：直接部署

1. **创建GitHub仓库**
   - 在GitHub上创建一个新的仓库
   - 仓库名称可以是任意的

2. **上传文件**
   - 将本项目的所有文件（index.html, styles.css, app.js, README.md）上传到仓库

3. **配置GitHub Pages**
   - 进入仓库的Settings页面
   - 滚动到Pages部分
   - 在Source选项中选择`main`分支或`master`分支
   - 选择`/(root)`作为目录
   - 点击Save按钮
   - 等待几分钟后，网站将在`https://[用户名].github.io/[仓库名]`上线

### 方法二：使用Git命令行

```bash
# 初始化仓库
git init
# 添加远程仓库
git remote add origin https://github.com/[用户名]/[仓库名].git
# 添加所有文件
git add .
# 提交更改
git commit -m "Initial commit"
# 推送到GitHub
git push -u origin main
# 然后按照方法一的步骤3配置GitHub Pages
```

## 题库格式说明

请按照以下格式创建TXT题库文件：

```
1. 题目内容（ ）。
A. 选项A
B. 选项B
C. 选项C
D. 选项D
答案：A

2. 多选题内容（ ）。
A. 选项A
B. 选项B
C. 选项C
D. 选项D
答案：AD
```

- 题目必须以`数字.`开头
- 选项必须以`A.`, `B.`, `C.`, `D.`开头
- 答案必须以`答案：`开头，多选题答案如`AD`（不要有空格）

## 使用说明

1. **上传题库**：点击"上传题库"按钮，选择符合格式的TXT文件
2. **选择题库**：在已上传的题库列表中选择要练习的题库
3. **开始练习**：点击"开始练习"进入答题界面
4. **答题操作**：
   - 单选题：点击选项选择答案
   - 多选题：点击多个选项选择答案
   - 点击"检查答案"验证正确性
   - 点击"查看答案"直接显示正确答案
5. **错题复习**：在首页点击"错题练习"或练习完成后点击"复习错题"

## 数据存储

所有数据均存储在浏览器的LocalStorage中：
- 上传的题库
- 用户答题记录
- 错题集
- 学习进度和称号

**注意**：清除浏览器缓存或使用隐私模式会导致数据丢失。不同用户在不同设备/浏览器上的数据完全独立，互不影响。
